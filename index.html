GitHubを使った公開フロー

公開サイトの構造は `src/` が決め、掲載内容は `content/` が決めます。編集、承認、履歴、自動公開をGitHub上で一元管理します。

1. 編集CMS画面またはGitHubでMarkdown / JSON / 画像を編集
2. PR作成変更差分をPull Requestとして提出
3. 自動チェックビルド、リンク、画像サイズ、アクセシビリティを確認
4. レビュー学科・担当者が内容を確認して承認
5. マージmainブランチに取り込み、正本を更新
6. 自動公開GitHub Actionsで静的サイトを生成して公開
このプロトタイプではHTMLに直接表示を記述していますが、実運用ではAstroなどが `content/` のファイルを読み込み、`src/pages/` と `src/components/` に従ってHTMLを自動生成します。
GitベースCMS運用プロトタイプ。編集対象は content/ と public/images/、構造設計は src/ に分離します。
